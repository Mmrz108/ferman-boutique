# فرمان (FERMAN) — بوتیک لوکس پوشاک زنانه و مردانه

پروژه Next.js + TypeScript + Tailwind + Framer Motion + React Three Fiber با یک **هیروی سه‌بعدی متصل به اسکرول واقعی** در صفحه اصلی.

---

## مرحله ۱ — خلاصه معماری

### چرا این ساختار؟

مشکل اصلی در ساخت یک هیروی سه‌بعدی واقعاً وابسته به اسکرول این است که اگر مقدار پیشرفت اسکرول را در **state ری‌اکت** ذخیره کنیم، هر پیکسل اسکرول باعث رندر مجدد کل درخت کامپوننت می‌شود — که برای یک صحنه‌ی WebGL در ۶۰ فریم بر ثانیه فاجعه‌بار است.

راه‌حل: مقدار اسکرول در یک **ref** نگه داشته می‌شود، نه state. صحنه‌ی سه‌بعدی در `useFrame` خودش (که مستقل از چرخه‌ی رندر ری‌اکت اجرا می‌شود) هر فریم این ref را می‌خواند و موقعیت/چرخش/دوربین را بر همان اساس محاسبه می‌کند. نتیجه: حرکت سه‌بعدی ۱۰۰٪ تابعی از موقعیت اسکرول است، نه از زمان، و هیچ رندر اضافه‌ای هم رخ نمی‌دهد.

### جریان داده (Data Flow)

```
کاربر اسکرول می‌کند
        │
        ▼
useScroll (framer-motion) روی <section ref={sectionRef}>
        │  scrollYProgress: MotionValue<number> از ۰ تا ۱
        ▼
useMotionValueEvent("change") → progressRef.current = latest
        │  (بدون setState — بدون رندر مجدد)
        ▼
   ┌────┴─────────────────────────┐
   ▼                              ▼
HeroOverlay                  Hero3DScene
useTransform(scrollYProgress)     useFrame(() => {
→ opacity / y / blur متن              t = progressRef.current
  (این بخش از MotionValue           rotation/position/camera
   مستقیم استفاده می‌کند،            بر اساس mapRange(t, ...)
   چون فقط CSS تغییر می‌کند)      })
```

دو مسیر متفاوت عمداً انتخاب شده‌اند:
- **HeroOverlay** مستقیماً از `scrollYProgress` (MotionValue) با `useTransform` استفاده می‌کند، چون این فقط CSS را آپدیت می‌کند و فریمر موشن خودش بهینه است.
- **Hero3DScene** از یک **ref ساده** استفاده می‌کند چون باید داخل حلقه‌ی رندر three.js (`useFrame`) خوانده شود؛ به MotionValue مستقیم دسترسی ندارد.

### معماری کامپوننت‌ها

```
ScrollDrivenHero (موتور اصلی بخش هیرو)
├── useHeroScrollProgress(sectionRef)   ← هوک مشترک: ref اسکرول + MotionValue
├── <section h-[300vh]>                 ← ارتفاع زیاد = فضای کافی برای روایت اسکرول
│   └── <div sticky top-0 h-screen>     ← صحنه روی صفحه ثابت می‌ماند
│       ├── HeroCanvasLazy              ← dynamic import، ssr: false
│       │   └── HeroCanvas              ← تنظیمات Canvas، دوربین، فاگ، محیط
│       │       └── Hero3DScene         ← مش روبان + نور + transforms در useFrame
│       │           └── ribbonGeometry  ← ساخت هندسه‌ی نوار از یک منحنی Catmull-Rom
│       └── HeroOverlay                 ← متن فارسی + دکمه‌ها، با useTransform محو می‌شوند
```

### مراحل حرکت سه‌بعدی (مطابق با `t = scrollYProgress` از ۰ تا ۱)

| بازه t | رفتار |
|---|---|
| ۰٫۰۰ – ۰٫۲۵ | روبان در مرکز، چرخش آرام پایه، متن کامل دیده می‌شود |
| ۰٫۲۵ – ۰٫۷۰ | چرخش فعال در دو محور، جابه‌جایی به چپ/عقب، نور کلیدی زاویه عوض می‌کند، متن محو و کمی بالا می‌رود |
| ۰٫۷۰ – ۱٫۰۰ | روبان به بالا/عقب می‌رود، کوچک و کم‌رنگ می‌شود تا فضا برای بخش کالکشن‌ها باز شود |

تمام این مقادیر با تابع کمکی `mapRange(t, inMin, inMax, outMin, outMax)` در `src/lib/useHeroScrollProgress.ts` محاسبه می‌شوند — یک نگاشت خطی محدودشده، دقیقاً مثل `interpolate` در فریمر موشن اما برای استفاده داخل `useFrame`.

### چرا یک «روبان» سه‌بعدی؟

به‌جای یک پرایمیتیو ساده (کره، مکعب) یا یک مدل واقعی لباس (که سفارش‌سازی و سبک نگه‌داشتنش سخت است)، یک **هندسه‌ی روبان** به‌صورت برنامه‌نویسی‌شده ساخته شده: یک منحنی Catmull-Rom در فضای سه‌بعدی که حول یک محور می‌پیچد و باد می‌کند (شبیه افت پارچه)، و سپس یک نوار صاف و نازک (نه لوله) دقیقاً روی قاب فرنه‌ی آن منحنی کشیده می‌شود. این فرم انتزاعی است اما به‌طور هم‌زمان به افتادگی پارچه (زنانه) و خط دقیق (مردانه) اشاره می‌کند، بدون اینکه شبیه لباس مشخصی باشد — دقیقاً همان ترکیب بصری خنثی-اما-لوکس که برای بوتیکی با دو کالکشن لازم است.

### استراتژی RTL

- `<html lang="fa" dir="rtl">` در `layout.tsx` — این تنها و کافی‌ترین نقطه‌ی تنظیم RTL است؛ کل صفحه بدون نیاز به mirror کردن دستی هر کلاس Tailwind به‌درستی راست‌به‌چپ می‌شود.
- چیدمان‌ها از پایه برای RTL طراحی شده‌اند (نه اینکه از یک طرح انگلیسی mirror شده باشند): مثلاً در هیرو، متن از سمت راست شروع می‌شود و فضای بصری در سمت چپ برای حرکت روبان باز گذاشته شده — این انتخاب، نه یک mirror خام.
- فونت **Vazirmatn** (از طریق `next/font/google`) چون برای فارسی بهینه‌سازی شده و چندین وزن (۳۰۰ تا ۸۰۰) برای ساخت سلسله‌مراتب تایپوگرافی دارد.

### عملکرد (Performance)

- صحنه‌ی سه‌بعدی با `dynamic(..., { ssr: false })` به‌صورت lazy لود می‌شود — باندل اولیه سبک می‌ماند و three.js هرگز در SSR اجرا نمی‌شود.
- `dpr` (device pixel ratio) محدود به `[1, 1.5]` در موبایل و `[1, 2]` در دسکتاپ.
- تعداد سگمنت‌های هندسه‌ی روبان در موبایل کمتر است (۱۲۰ به‌جای ۲۲۰).
- `prefers-reduced-motion` به‌صورت کامل احترام گذاشته می‌شود: هم در CSS (`globals.css`) و هم در صحنه‌ی سه‌بعدی (چرخش idle و حرکت‌های ریز خاموش می‌شوند، فقط حرکت اسکرول‌محور باقی می‌ماند).
- المان‌های نامرئی هیرو (بعد از محو شدن) با `pointer-events: none` و `aria-hidden` از مسیر فوکوس کیبورد و screen reader خارج می‌شوند.

## مرحله ۴ — راه‌اندازی پروژه

```bash
# ۱) از فایل zip خارج کنید و وارد پوشه شوید
cd ferman

# ۲) نصب وابستگی‌ها
npm install

# ۳) اجرای محیط توسعه
npm run dev

# سایت روی http://localhost:3000 در دسترس است
```

برای build production:

```bash
npm run build
npm run start
```

> **نکته:** پروژه از `next/image` با تصاویر نمونه از Unsplash استفاده می‌کند (`images.unsplash.com` در `next.config.js` به‌عنوان `remotePatterns` تنظیم شده). برای استفاده در پروداکشن واقعی، این تصاویر را با عکس‌های واقعی محصولات خودتان جایگزین کنید.

## مرحله ۵ — راهنمای سفارشی‌سازی

### تغییر متن‌های فارسی
همه‌چیز در **`src/data/content.ts`** است — یک فایل واحد برای:
- `hero` → عنوان، زیرنویس، متن دکمه‌ها، متن "برای ادامه پیمایش کنید"
- `nav` → آیتم‌های منو
- `collections` → عنوان/توضیح/تصویر هر کارت کالکشن
- `products` → نام، دسته، قیمت، تصویر هر محصول
- `story` → متن بخش درباره‌ی برند
- `lookbook`, `newsletter`, `footer` → به همین ترتیب

هیچ‌جای دیگری در کد متن فارسی hardcode نشده — تغییر این فایل کافی است.

### تغییر رنگ‌ها
در **`tailwind.config.ts`** بخش `colors`:
```ts
ink: "#0E0D0C",        // پس‌زمینه‌ی اصلی (مشکی ذغالی)
panel: "#1A1815",      // سطوح کارت‌ها
cream: "#F5EEE3",      // متن روشن / دکمه‌ها
champagne: "#C9A063",  // رنگ تاکیدی طلایی — به‌عمد محدود استفاده شده
taupe: "#8A8178",      // متن ثانویه
```
رنگ متریال روبان سه‌بعدی جدا تنظیم می‌شود (پایین را ببینید).

### تغییر داده‌ی محصولات/کالکشن‌ها
آرایه‌های `products` و `collections` در `content.ts` — هر آیتم را اضافه/حذف/ویرایش کنید؛ گرید و کارت‌ها به‌صورت خودکار با تعداد آیتم‌های جدید هماهنگ می‌شوند.

### تغییر محتوای هیرو
آبجکت `hero` در `content.ts` — `eyebrow`, `headline`, `subtext`, و سه دکمه‌ی `ctas`.

### تغییر رفتار سه‌بعدی روبان

در **`src/components/hero/Hero3DScene.tsx`**، داخل `useFrame`، هر خط با `mapRange(t, ...)` یک پارامتر حرکتی است:

```ts
const rotY = mapRange(t, 0, 1, 0.15, Math.PI * 1.65);
//                        ↑    ↑    ↑      ↑
//                   شروع t  پایان t  مقدار شروع  مقدار پایان
```

برای تغییر شدت/زمان‌بندی هر حرکت (چرخش، جابه‌جایی X/Y/Z، تغییر دوربین، نور)، همین اعداد را ویرایش کنید. برای مثال اگر می‌خواهید چرخش زودتر شروع شود، بازه‌ی `inMin/inMax` را کوچک‌تر کنید (مثلاً از `0, 1` به `0, 0.5`).

برای تغییر **شکل خود روبان** (پیچش، شعاع، ارتفاع)، به **`src/components/hero/ribbonGeometry.ts`** بروید — تابع `createRibbonCurve` پارامترهای `radius`, `height`, و زاویه‌ی پیچش (`Math.PI * 2.4`) را کنترل می‌کند. برای تغییر پهنا/ضخامت نوار، آرگومان‌های `ribbonWidth` و `ribbonThickness` در `buildRibbonGeometry`.

رنگ و جلای متریال (طلایی/براق/مات) در `Hero3DScene.tsx` داخل `useMemo` ساخت `MeshPhysicalMaterial` قابل تنظیم است (`color`, `metalness`, `roughness`, `sheenColor`).

### تنظیم ارتفاع بخش اسکرول هیرو

در **`ScrollDrivenHero.tsx`**، کلاس `h-[300vh]` ارتفاع کل ناحیه‌ی اسکرول است. عدد بزرگ‌تر = روایت کش‌دارتر و آرام‌تر؛ عدد کوچک‌تر = حرکت سریع‌تر و فشرده‌تر.

### تغییر فونت
در **`src/app/layout.tsx`** فونت `Vazirmatn` از `next/font/google` ایمپورت شده. برای تغییر به فونت دیگر (مثلاً Yekan Bakh یا یک فونت سفارشی محلی)، این ایمپورت و متغیر `vazir` را با فونت دلخواه جایگزین کنید و `--font-vazir` در `tailwind.config.ts` را متناظر نگه دارید.

## نکات شناخته‌شده برای توسعه‌ی بعدی

- صفحات داخلی کالکشن‌ها (`/collections/[id]`) و صفحه‌ی محصول هنوز ساخته نشده‌اند — این یک ساخت تک‌صفحه‌ی Homepage است. لینک‌های مربوطه فعلاً placeholder (`#`) هستند.
- آیکون جستجو و کیف خرید در ناوبار صرفاً نمایشی‌اند و به یک کشوی جستجو/سبد خرید واقعی وصل نیستند.
- فرم خبرنامه فقط یک state محلی برای نمایش پیام موفقیت دارد و به هیچ سرویس ایمیل واقعی متصل نیست.
